### 在vue项目中,编译时和运行时是两个明确分离的阶段,他们分别发生在项目构建和浏览器执行的过程中

# 编译时(Compile-time)
    compile发生在项目的构建阶段,具体场景为:npm run build / yarn build / npm run start 的时候,你的源代码(.vue单文件组件中的模板)会被构建工具(vite/webpack)处理.

--------
| 核心任务 | 执行环境  | 目的与优势 | 典型场景 |
|:--------| :---------:|--------:|--------:|
| centered 将你编写的模板(template)代码,例如`<div>{{message}}</div>`编译成高性能的渲染render函数. | centered node.js环境中进行,而不是浏览器 |centered 通过提前编译,可以进行静态分析和优化(如标记静态节点),从而生成更高效的代码,这能显著减小最终打包文件的体积(因为不需要包含完整的编译器,比如一个compile.js模块)|centered 当你使用Vue-cli/webpack/vite等构建工具时,模板编译就发生在这一阶段.这是Vue官方在生产环境中强烈推荐的方式|
--------

# 运行时(Runtime)
    运行时发生在应用被部署到浏览器并加载执行之后.
--------
| 核心任务 | 执行环境  | 目的与优势 | 典型场景 |
|:--------| :---------:|--------:|--------:|
| centered 执行编译时生成的渲染函数,创建组件实例,管理响应式数据,并将虚拟DOM的变化精准更新到真实的浏览器DOM上 | centered 这个过程是在用户的浏览器环境中进行的 |centered 运行时核心是"执行"和"更新",他负责处理数据变化/事件响应和DOM操作,是用户与应用交互的直接载体,由于编译时已经完成了模板解析,运行时的包体积极小,启动和更新速度更快 |centered 当用户在浏览器中打开你的网站,看到的内容,点击按钮,数据更新导致液面变化时,这些都发生在运行时 |

所以Vue中有很多包
数据响应式/compile/runtime主包
